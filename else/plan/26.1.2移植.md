## ❌ 剩余工作

### 一、客户端 GUI 渲染系统重写（最大块，~30 个文件）

26.1 Mojang 把 GUI 从 immediate-mode 改成 **record-state-then-render**：
- `GuiGraphics` → `GuiGraphicsExtractor`（不只是改名，语义不同：从画笔变成 state 收集器）
- 各 widget 必须实现新的 `extractRenderState(GuiGraphicsExtractor, ...)` 而非旧的 `renderWidget(GuiGraphics, ...)`
- `BlockEntityRenderer<T>` 改成 `BlockEntityRenderer<T, RenderState>`，每个 BE 要自带 RenderState 类
- `render()` 方法被 `submit(RenderState, PoseStack, SubmitNodeCollector, CameraRenderState)` 替代

**影响文件清单**（22 个 client 包）：
- Renderer (4)：[CrystalCatalyzerRenderer.java](src/main/java/com/moakiee/ae2lt/client/CrystalCatalyzerRenderer.java)、[FumoBlockRenderer.java](src/main/java/com/moakiee/ae2lt/client/FumoBlockRenderer.java)、[LightningAssemblyChamberRenderer.java](src/main/java/com/moakiee/ae2lt/client/LightningAssemblyChamberRenderer.java)、[LightningSimulationChamberRenderer.java](src/main/java/com/moakiee/ae2lt/client/LightningSimulationChamberRenderer.java)

## 风险点

- **GUI 重构是 Mojang 26.1 最大变化**——参考资料只有 ae2-26.1 源码本身，没有完整迁移指南
- **AppFlux/Mekanism/AdvancedAE/NeoEcoAE/ExtendedAE 都没出 26.1.2**——所有相关集成代码注定要 stub 化或长期禁用
- **mixin 必然要打开游戏才能验证**——编译过了不代表注入成功
- **datagen / 资源**：`src/generated/resources/` 内容是 Phase 1.21.1 时机的产物，26.1.2 后可能要重跑 `runData`，但 datagen 本身依赖 GUI/render 类，要等客户端编译通才能跑
