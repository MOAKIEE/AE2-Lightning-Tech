# 과부하 ME 인터페이스

<Row>
  <BlockImage id="ae2lt:overloaded_interface" scale="4" />
</Row>

**과부하 ME 인터페이스**는 바닐라 <ItemLink id="ae2:interface" />의 강화 버전으로, **36개의 설정 슬롯**을 가지고 있으며, **무선 모드**를 지원하여 원격 아이템 전송과 에너지 분배를 수행할 수 있습니다.

## 핵심 특성

* **36개의 설정 슬롯**(바닐라는 9개)
* **두 가지 작동 모드**: 일반 모드와 무선 모드
* **무한 모드 슬롯**: 단일 슬롯을 무한 공급으로 전환 가능
* **두 단계의 I/O 속도**: 일반과 고속(프로브)
* **에너지 전송**: ME 네트워크의 FE 에너지를 인접 또는 무선으로 연결된 기계로 출력 지원
* **입력 필터링**: 과부하 필터 요소를 통해 가져오기 허용 아이템 종류 제한

## 작동 모드

### 일반 모드

일반 모드에서 과부하 ME 인터페이스의 동작은 바닐라 인터페이스와 유사합니다 — 설정에 따라 자동으로 보충하고 아이템을 물리적으로 인접한 컨테이너로 출력합니다. 주요 차이점은 더 많은 설정 슬롯과 더 큰 전송 대역폭을 가진 것입니다.

에너지 출력 방향이 설정된 경우, 일반 모드에서도 ME 네트워크의 FE 에너지를 지정된 방향의 인접 설비로 출력할 수 있습니다.

### 무선 모드

무선 모드에서 과부하 ME 인터페이스는 여러 기계의 입력, 출력 및 에너지 공급을 원격 관리할 수 있습니다.

**과부하 무선 연결 도구**를 사용하여 무선 연결을 설정:

1. 도구를 들고 과부하 ME 인터페이스에 **Shift + 우클릭**으로 선택
2. 다음 대상 기계의 특정 면을 **우클릭**으로 연결 설정
3. 동일한 인터페이스는 여러 원격 기계를 바인딩 가능

## 입력 / 출력 모드

| 모드 | 설명 |
|------|------|
| 출력: 끔 | 원격 기계로 아이템 출력 안 함 |
| 출력: 자동 | 설정 슬롯에 따라 원격 기계로 자동 출력 |
| 입력: 끔 | 원격 기계에서 아이템 가져오기 안 함 |
| 입력: 자동 | 원격 기계에서 ME 네트워크로 아이템 지속 가져오기 |
| 입력: 추출 | 원격 기계가 가상 입력 포트로 아이템을 밀어넣음; 인터페이스는 수동 수신 |

## I/O 속도

| 속도 단계 | 적응형 쿨다운 범위 | 설명 |
|-------|--------------|------|
| 일반 | 5 ~ 80 tick | 일반적인 시나리오에 적합 |
| 고속(프로브) | 1 ~ 40 tick | 프로브 메커니즘으로 준비 상태를 미리 감지; 가속 카드 장착 후 사용 가능 |

인터페이스는 적응형 쿨다운을 사용 — 전송 가능한 아이템이 있을 때 쿨다운 시간이 자동으로 단축되며, 작업이 없을 때 쿨다운 시간이 자동으로 연장되어 성능과 응답 사이에서 균형을 잡습니다.

## 무한 모드

설정 슬롯을 Shift + 클릭하면 **무한 모드**로 전환할 수 있습니다. 무한 모드에서는 해당 슬롯이 설정된 아이템을 원격 기계로 무한 수량으로 지속 공급합니다.

## 입력 필터링

<ItemImage id="ae2lt:overloaded_filter_component" scale="2" float="left" />

과부하 ME 인터페이스는 **과부하 필터 요소** 장착을 지원하여 ME 네트워크로 가져오기 허용 아이템 종류를 제한합니다. 필터 요소의 설정 방법은 AE2 셀 작업대와 유사합니다.

## 에너지 전송

감응 카드(AppFlux 모드 지원 필요)를 장착한 후, 과부하 ME 인터페이스는 ME 네트워크에 저장된 FE 에너지를 대상 기계로 출력할 수 있습니다.

* **일반 모드**: 지정된 방향의 인접 블록으로 에너지 출력
* **무선 모드**: 무선으로 연결된 모든 기계로 에너지를 분배, 타임 라운드 로빈 스케줄러가 자동으로 할당

## 자동화 권장사항

* 단일 인터페이스로 여러 기계의 입력/출력을 무선으로 관리하여, 많은 양의 파이프를 대체 가능
* 무한 모드 슬롯과 조합하여 원격 기계에 특정 재료를 지속적으로 제공 가능
* 에너지 전송 기능과 결합하여 별도의 에너지 라인을 생략 가능
* 프로브 속도 단계를 활성화하여 더 낮은 가공 응답 지연 획득
